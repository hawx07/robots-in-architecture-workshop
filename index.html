1. What is Arduino?

Think of Arduino as the "brain" of your robot. It is a small computer (microcontroller) that can read information from sensors (like eyes) and send instructions to motors (like muscles).

2. The Arduino IDE

The IDE (Integrated Development Environment) is the software we use on our laptops to write code and "upload" it to the Arduino brain via a USB cable.

3. Anatomy of a "Sketch"

Every Arduino program has two main parts. Think of it like a recipe:

void setup() {
  // This happens ONLY ONCE when the robot starts.
  // Use this to "wake up" the motors or sensors.
}

void loop() {
  // This happens OVER AND OVER AGAIN forever.
  // This is where the "logic" of the robot lives.
}

4. Data "Knobs"

In your provided modular code, you will see variables. To change how the robot behaves, you will "tweak" these values:

  • int (Integers): Whole numbers. Used for speed or pin numbers.
  • float (Floating point): Numbers with decimals. Used for precise distances or angles.
  • bool (Boolean): Simple True or False (e.g., lights on/off).

5. Basic Syntax

Computers are very literal. If you miss a tiny symbol, the code won't run:

  • Semicolons ( ; ): Every instruction must end with a ;.
  • Curly Braces ( { } ): These group blocks of code together. If you open one, you MUST close it.
  • Comments ( // ): Notes for humans to explain what the code does; ignored by the robot.
