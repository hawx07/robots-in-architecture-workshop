Build order

Recommended Workflow

1. Test one part

Wire one sensor or actuator alone. Run the smallest sketch that proves it works.

2. Print values

Use Serial Monitor before moving motors. Know the real sensor range in the room.

3. Choose thresholds

Write down dry/wet, bright/dark, near/far, hot/cool values from the actual setup.

4. Add one actuator

Connect a servo, LED, buzzer, fan, or motor driver after the sensor is stable.

5. Combine slowly

Add the next sensor only after the previous behavior is predictable.

6. Mount and retest

Values change after parts are mounted inside models. Calibrate again.

Core concepts

Electricity Basics

ConceptMeaningWorkshop Rule
VoltageElectrical pressure, measured in volts.Use the voltage the module expects: 5V, 3.3V, or motor battery voltage.
CurrentHow much electricity flows, measured in amps.Motors and servos need much more current than sensors.
GroundThe common reference point for all signals.Separate supplies still need shared GND for signals to work.
ResistanceLimits current or forms voltage dividers.Use resistors with LEDs and LDR circuits.
Short circuitPower connected directly to ground.Disconnect immediately if parts heat up, smell, or reset the board.

Rules That Prevent Most Failures

  • Never power a DC motor or multiple servos directly from an Arduino pin.
  • Connect all grounds together when using separate power supplies.
  • Put a resistor in series with every bare LED.
  • Check polarity before powering modules: VCC to VCC, GND to GND.
  • Upload and test code before installing electronics deep inside a model.
  • Secure wires on anything that moves.

Making circuits

Breadboards and Wiring

Breadboard Layout

Rows are connected horizontally in the middle area. Power rails run vertically along the sides, but some boards split rails halfway, so test or bridge them if needed.

Color Convention

Use red for 5V or 3.3V, black or blue for GND, yellow/white/green for signal. Consistent colors make debugging faster.

PartTypical UseCommon Mistake

Inputs and outputs

Arduino Pin Types

Digital input

Reads HIGH or LOW. Use for buttons, PIR sensors, tilt switches, line sensors, and IR modules.

Digital output

Turns something on/off. Use for LEDs, buzzers, relays, and driver inputs.

Analog input

Reads 0-1023 on Arduino Uno. Use for LDR, potentiometer, joystick, sound, gas, rain, and temperature sensors.

PWM output

Simulates analog output with fast pulses. Use for LED dimming and DC motor speed through a driver.

I2C

Two-wire bus, usually SDA/SCL. Use for LCD modules and some sensor boards.

SPI

Fast multi-wire bus. Common for RC522 RFID modules.

Reliability

Power Planning

Separate Logic and Motion

Arduino USB can power the board and small sensors. Servos, fans, and motors should usually use a separate supply or the kit battery.

Share Ground

If the actuator supply is separate, connect its GND to Arduino GND. Without common ground, signal pins have no shared reference.

  • If the board resets when a servo moves, the power supply is too weak or wiring is poor.
  • If a motor does not move but the code runs, check driver power and enable pins.
  • If sensors become noisy when motors run, separate motor power wires from signal wires and add smoothing capacitors if available.

Reusable logic

Code Patterns

Threshold With Hysteresis

bool openState = false;

if (!openState && value > openThreshold) {
  openState = true;
}

if (openState && value < closeThreshold) {
  openState = false;
}

Analog Sensor Mapping

int raw = analogRead(A0);
int percent = map(raw, dryValue, wetValue, 0, 100);
percent = constrain(percent, 0, 100);

Button Toggle With INPUT_PULLUP

bool state = false;
bool lastButton = HIGH;

bool button = digitalRead(buttonPin);
if (button == LOW && lastButton == HIGH) {
  state = !state;
  delay(40);
}
lastButton = button;

Non-Blocking Timing

unsigned long lastUpdate = 0;

if (millis() - lastUpdate > 500) {
  lastUpdate = millis();
  readSensor();
  updateActuator();
}

When it fails

Debugging Checklist

  1. Does the board power LED turn on?
  2. Is the correct board and port selected in Arduino IDE?
  3. Does a simple blink sketch upload?
  4. Are VCC and GND correct on the module?
  5. Are all grounds connected when using separate power?
  6. Does Serial Monitor show sensor values?
  7. Are the real values crossing the threshold in your code?
  8. Can the actuator work in a separate minimal test sketch?
  9. Does the behavior still work after mounting in the model?

Workshop practice

Safety and Build Quality

  • Disconnect power before changing motor, servo, or battery wiring.
  • Keep fingers clear of robot arm joints, gears, and wheels during upload and reset.
  • Use low-voltage loads only unless supervised by qualified staff.
  • Do not leave batteries charging unattended.
  • Add strain relief to wires on moving parts.
  • Document final pin maps and thresholds directly in the sketch comments.
